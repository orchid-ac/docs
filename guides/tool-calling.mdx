## Overview

There are two kinds of tool use on orchid01:

- **Built-in tools** (`orchid-finance`) — run automatically on every request, on all three endpoints ([chat completions](/api-reference/chat-completions), [messages](/api-reference/messages), [responses](/api-reference/responses)). No setup required. See [Built-in tools](#built-in-tools-orchid-finance) below.
- **Your own tools** — defined with JSON Schema and passed in the request. Supported on **[`/v1/chat/completions`](/api-reference/chat-completions)** and **[`/v1/responses`](/api-reference/responses)**. The Anthropic **[`/v1/messages`](/api-reference/messages)** façade does not support tool-use content blocks yet — route custom tool workflows through one of the other two endpoints.

This works with any tool definition you'd use with OpenAI — including MCP tools, LangChain tools, and custom functions.

<Note>
  `stream: true` is silently overridden to a non-streamed response whenever a tool call happens mid-response — built-in or your own — on any of the three endpoints. The model has to finish its tool-call loop before it can return a response, so there's nothing to stream until that's done.
</Note>

## Built-in tools (orchid-finance)

orchid01 ships with a built-in tool suite, `orchid-finance`, that the model can call on its own to ground its answers — for example pulling live filing data — without you defining or passing any `tools`. This runs automatically on every request, including ones that pass no `tools` array at all.

### MCP audit fields

Every response includes an `orchid` block with audit fields describing what the built-in tool layer did:

```json
{
  "orchid": {
    "mcp_server":     "orchid-finance",
    "mcp_iterations": 2,
    "mcp_tool_calls": [
      {
        "tool":   "get_sec_filing",
        "args":   { "ticker": "AAPL", "form_type": "10-K" },
        "result": "Apple Inc. 10-K filed 2025-11-01 (preview, truncated)..."
      }
    ]
  }
}
```

| Field | Description |
|-------|--------------|
| `mcp_server` | Which tool server handled the call (e.g. `orchid-finance`) |
| `mcp_iterations` | Number of tool-call rounds the model took before responding |
| `mcp_tool_calls` | List of tools called, their arguments, and a preview of each result |

These fields are always present, separate from any `tools` you pass yourself and from the `dehallucinate`/`regenerate` fields described in [Hallucination detection](/guides/hallucination-detection).

## Basic example

```python
tools = [{
    "type": "function",
    "function": {
        "name":        "get_sec_filing",
        "description": "Retrieve an SEC filing by ticker and form type",
        "parameters": {
            "type": "object",
            "properties": {
                "ticker":    {"type": "string", "description": "Company ticker symbol"},
                "form_type": {"type": "string", "enum": ["10-K", "10-Q", "8-K", "13-F"]},
            },
            "required": ["ticker", "form_type"],
        },
    },
}]

response = client.chat.completions.create(
    model="orchid01",
    messages=[{"role": "user", "content": "Get Apple's latest 10-K"}],
    tools=tools,
    temperature=0.1,
)

tool_call = response.choices[0].message.tool_calls[0]
print(tool_call.function.name)       # get_sec_filing
print(tool_call.function.arguments)  # {"ticker": "AAPL", "form_type": "10-K"}
```

## Full agentic loop

```python
import json

messages = [{"role": "user", "content": "Analyse Apple's latest 10-K for covenant risks"}]

response = client.chat.completions.create(
    model="orchid01",
    messages=messages,
    tools=tools,
)

if response.choices[0].message.tool_calls:
    tool_call = response.choices[0].message.tool_calls[0]
    args      = json.loads(tool_call.function.arguments)
    
    # Execute your tool
    result = get_sec_filing(**args)
    
    # Send result back
    messages.append(response.choices[0].message)
    messages.append({
        "role":         "tool",
        "tool_call_id": tool_call.id,
        "content":      json.dumps(result),
    })
    
    # Get final response
    final = client.chat.completions.create(
        model="orchid01",
        messages=messages,
        tools=tools,
        extra_body={"orchid": {"dehallucinate": True}},
    )
    print(final.choices[0].message.content)
```

## Your own MCP tools

orchid01 also works with MCP (Model Context Protocol) tools you bring yourself — this is separate from the built-in `orchid-finance` suite above. Your MCP server runs client-side — translate its tool definitions into JSON Schema format and pass them in the request as shown above.

Popular finance MCP servers that work with orchid01:

- **EdgarTools** — 13 tools for SEC EDGAR filings, financial statements, insider trading, live filings feed. No API key required.
- **Bloomberg** — connects to your existing Bloomberg Terminal via blpapi.
- **Alpha Vantage** — real-time and historical market data.

## LangChain agents

```python
from langchain_openai  import ChatOpenAI
from langchain.agents  import AgentExecutor, create_openai_tools_agent
from langchain.tools   import tool

llm = ChatOpenAI(
    model="orchid01",
    base_url="https://llm.orchid.ac/v1",
    api_key="orchid-your-key-here",
    temperature=0.1,
)

@tool
def get_covenant_data(company: str) -> str:
    """Get covenant data for a company from internal database"""
    # your implementation
    ...

agent = create_openai_tools_agent(llm, [get_covenant_data], prompt)
executor = AgentExecutor(agent=agent, tools=[get_covenant_data])
executor.invoke({"input": "Check covenant headroom for Citadel"})
```

## Tips

- Use low temperature (`0.0–0.2`) for tool calling — reduces hallucinated parameter values
- Write detailed tool descriptions — the model decides when to call a tool based on the description
- Use `tool_choice="required"` to force the model to always call a tool
