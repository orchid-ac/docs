## Overview

On **non-streaming** [`/v1/chat/completions`](/api-reference/chat-completions) requests, you can opt in to a **post-reply grounding judge** by setting **`orchid.dehallucinate`** to **`true`**. On **[`/v1/messages`](/api-reference/messages)**, send **`x-orchid-dehallucinate: true`**.

By default **`dehallucinate` is `false`** ā€” the judge does not run unless you enable it. When it runs and enough document context is present, the response includes a grounding score: how well the model's answer is supported by the materials you provided.

## How it works

The grounding check compares the model's response against your provided context ā€” everything in `messages` except the last user message. It identifies specific claims that cannot be verified against your documents.

```
Your messages (system + prior turns) ā†’ context
Last user message                    ā†’ question  
Model response                       ā†’ checked against context
```

If a response contains a financial figure that doesn't appear in the provided context, it is flagged as a potentially unsupported claim.

## Enabling the check

**Chat completions** ā€” set `dehallucinate` to `true` in the `orchid` object:

```python
response = client.chat.completions.create(
    model="orchid01",
    messages=[...],
    extra_body={"orchid": {"dehallucinate": True}},
)
```

**Messages API** ā€” send header `x-orchid-dehallucinate: true` (see [Messages API](/api-reference/messages)).

## Reading the grounding score

When the judge runs, the response includes an `orchid` metadata field:

```json
{
  "choices": [...],
  "orchid": {
    "dehallucinate_requested": true,
    "grounded":                true,
    "score":                   0.97,
    "flagged_spans":           [],
    "checked":                 true
  }
}
```

| Field | Description |
|-------|-------------|
| `grounded` | `true` if no unsupported claims were found |
| `score` | 0.0ā€“1.0. Higher is better. 1.0 = fully grounded |
| `flagged_spans` | List of specific text spans that could not be verified |
| `checked` | Whether the check actually ran |

## When the check is skipped

The check returns `checked: false` (and a default score of `1.0`) when:

- `dehallucinate` / `x-orchid-dehallucinate` was not enabled
- The response contains no financial figures
- The context is too short (under ~200 characters) ā€” likely a bare question with no documents

```json
{
  "orchid": {
    "checked": false,
    "reason":  "no financial figures in response",
    "score":   1.0
  }
}
```

## Leaving the judge off (default)

The default is **`dehallucinate: false`** for lower latency when you do not need verification on that call:

```python
response = client.chat.completions.create(
    model="orchid01",
    messages=[...],
    extra_body={"orchid": {"dehallucinate": False}},
)
```

Omitting `orchid` or omitting `dehallucinate` behaves like `false`.

<Note>
  The grounding judge only runs on **non-streaming** requests ā€” the full reply must be available before verification. Streaming responses return `checked: false` for this path.
</Note>

## Best practice

For the most accurate grounding scores, provide document context explicitly in your messages rather than relying on the model's parametric memory:

```python
messages = [
    {
        "role": "system",
        "content": "You are a financial analyst assistant.",
    },
    {
        "role": "user",
        # Provide the actual document text
        "content": f"Filing excerpt:\n\n{filing_text}\n\nWhat was Q3 revenue?",
    }
]
```

Without document context, the model answers from memory and the grounding check has nothing to verify against.
