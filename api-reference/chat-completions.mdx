## Endpoint

```
POST https://llm.orchid.ac/v1/chat/completions
```

OpenAI-compatible. Accepts the same request body as `POST /v1/chat/completions` on the OpenAI API, plus an optional `orchid` config object.

## Request

### Headers

| Header | Value |
|--------|-------|
| `Authorization` | `Bearer <your-orchid-key>` |
| `Content-Type` | `application/json` |

### Body

All standard OpenAI chat completion fields are supported. Key fields:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `model` | string | required | Use `orchid01` |
| `messages` | array | required | Conversation history |
| `max_tokens` | integer | `4096` | See [Max Tokens](/guides/max-tokens) |
| `temperature` | number | `0.1` | Ignored in thinking mode (fixed at 1.0) |
| `stream` | boolean | `false` | Stream response tokens |
| `tools` | array | — | Tool definitions (JSON Schema) |
| `tool_choice` | string/object | `auto` | Tool selection behaviour |
| `orchid` | object | — | Orchid-specific config (see below) |

### The `orchid` config object

Optional body key **`orchid`** controls Orchid-specific behaviour on **`/v1/chat/completions`** only (not on [`/v1/messages`](/api-reference/messages)).

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `thinking` | boolean | `false` | Enable deep reasoning mode (Fireworks) |
| `thinking_effort` | string | — | When thinking is on: `none`, `low`, `medium`, or `high` |
| `dehallucinate` | boolean | `false` | Set `true` to run the post-reply grounding judge on **non-streaming** completions |
| `regenerate` | boolean | `false` | Set `true` to auto-correct unsupported claims where possible. Implies `dehallucinate: true`. |

See [Hallucination detection](/guides/hallucination-detection) and [Rate limits](/api-reference/rate-limits).

<Note>
  The built-in `orchid-finance` tool suite runs automatically on every request (see [Tool calling](/guides/tool-calling)). If it — or a `tools` array you passed — triggers a tool call, `stream: true` is overridden to a non-streamed response: the model must finish its tool-call loop before it can return anything.
</Note>

## Response

Standard OpenAI chat completion response, plus an `orchid` field:

```json
{
  "id":      "chatcmpl-abc123",
  "object":  "chat.completion",
  "model":   "orchid01",
  "choices": [{
    "index":         0,
    "message": {
      "role":    "assistant",
      "content": "According to the filing..."
    },
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens":     1240,
    "completion_tokens": 380,
    "total_tokens":      1620
  },
  "orchid": {
    "mcp_server":              "orchid-finance",
    "mcp_iterations":          1,
    "mcp_tool_calls":          [],
    "dehallucinate_requested": true,
    "grounded":                true,
    "score":                   0.97,
    "flagged_spans":           [],
    "checked":                 true
  }
}
```

`mcp_server`, `mcp_iterations`, and `mcp_tool_calls` are always present — see [Tool calling](/guides/tool-calling#mcp-audit-fields). The `dehallucinate_requested`, `grounded`, `score`, `flagged_spans`, and `checked` fields only appear with their documented values when `dehallucinate` (or `regenerate`) was enabled — see [Hallucination detection](/guides/hallucination-detection).

## Error responses

**Format:** Failures return an **OpenAI-compatible** JSON body with a top-level `error` object (`message`, `type`, `code`, `param`). The same shape is used for gateway errors across Orchid so OpenAI SDKs parse errors predictably.

Errors follow that format, for example:

```json
{
  "error": {
    "message": "Invalid API key",
    "type":    "authentication_error",
    "code":    "invalid_api_key",
    "param":   null
  }
}
```

| Status | Type | Meaning |
|--------|------|---------|
| `400` | `invalid_request_error` | Malformed request body |
| `401` | `authentication_error` | Invalid or missing API key |
| `429` | `rate_limit_error` | Rate limit exceeded — see [Rate limits](/api-reference/rate-limits) |
| `502` | `upstream_error` | Upstream provider error |
