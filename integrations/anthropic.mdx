Point the Anthropic SDK at Orchid’s origin with the same **`/v1`** base path as OpenAI-style clients. Use your **Orchid** API key for `api_key` — not a key from the Anthropic console.

**Errors:** Orchid returns **OpenAI-compatible** `error` JSON for many failures (including auth and rate limits), even on **`/v1/messages`**. The Anthropic SDK may surface confusing exceptions — check **raw HTTP** responses when debugging. See [Messages API](/api-reference/messages).

## Install

```bash
pip install anthropic
```

## Non-streaming

```python
import anthropic

client = anthropic.Anthropic(
    base_url="https://llm.orchid.ac/v1",
    api_key="orchid-your-key-here",
)

msg = client.messages.create(
    model="orchid01",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello"}],
)
print(msg.content[0].text)
```

## Streaming

```python
with client.messages.stream(
    model="orchid01",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello"}],
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)
```

## Optional Orchid headers

The Anthropic Python client accepts **`default_headers`** for Orchid extensions:

```python
import os
import anthropic

client = anthropic.Anthropic(
    base_url="https://llm.orchid.ac/v1",
    api_key=os.environ["ORCHID_API_KEY"],
    default_headers={
        "x-orchid-dehallucinate": "true",
        # "x-orchid-thinking": "true",
        # "x-orchid-thinking-effort": "medium",
    },
)
```

Alternatively, send the same headers with `curl` or any HTTP client.

## Claude Code and Anthropic-native tools

Products that expect **Anthropic** `tool_use` or **images** on **`/v1/messages`** may be unsupported on this façade (text-first). Use [`/v1/chat/completions`](/api-reference/chat-completions) for full OpenAI-style tools and multimodal content.

Configure the same **`base_url`** (`https://llm.orchid.ac/v1`), **`model: orchid01`**, and Orchid key (`x-api-key` or Bearer per [Authentication](/getting-started/authentication)).
