OpenAI's [Codex CLI](https://github.com/openai/codex) supports custom model providers through a TOML config file.

<Note>
  Codex CLI uses the **OpenAI Responses API** wire format (`wire_api = "responses"`).
</Note>

## Setup

**1. Add the Orchid provider to `~/.codex/config.toml`:**

```toml
model_provider = "orchid"
model          = "orchid01"

[model_providers.orchid]
name     = "Orchid"
base_url = "https://llm.orchid.ac/v1"
wire_api = "responses"
env_key  = "ORCHID_API_KEY"
```

**2. Export your Orchid API key:**

```bash
export ORCHID_API_KEY="orchid-your-key-here"
```

**3. Run Codex:**

```bash
codex "your prompt here"
```

## Usage

With `model_provider` and `model` set in `~/.codex/config.toml`, every `codex` invocation routes through Orchid automatically.

To override on a per-run basis:

```bash
codex -c model_provider="orchid" -m orchid01 "your prompt here"
```
