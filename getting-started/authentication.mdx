## API keys

Get your key from [app.orchid.ac](https://app.orchid.ac) under **Settings ‚Üí API Keys**. The secret string is the same whether you call OpenAI-style or Anthropic-style endpoints.

<Warning>
  Your API key is secret. Never commit it to version control or expose it in client-side code. Use environment variables.
</Warning>

## Connection settings

| Setting | Value |
|---------|-------|
| **Model** | `orchid01` ‚Äî the JSON `model` field must be this catalog id (see [GET /v1/models](/api-reference/models)) |
| **Responses** | Successful bodies and webhooks expose `model: "orchid01"` (not upstream provider ids) |

### Base URL (OpenAI-compatible SDKs and HTTP)

Use this for the **OpenAI** Python/JS SDK, Cursor, and any client that calls paths such as `/v1/chat/completions` under a configurable prefix.

| | Production | Local |
|---|------------|--------|
| **Base URL** | `https://llm.orchid.ac/v1` | `http://127.0.0.1:8000/v1` |

<Note>
  The OpenAI-style base URL **must** include **`/v1`**. If you use only the host (for example `https://llm.orchid.ac` with no path), requests that expect the `/v1/...` prefix will not resolve correctly.
</Note>

### Base URL (Anthropic SDK and Claude Code)

For the **Anthropic** Python/TypeScript SDK and **Claude Code** (`ANTHROPIC_BASE_URL`), set the base to the **host with no `/v1` suffix**:

| | Production | Local |
|---|------------|--------|
| **Base URL** | `https://llm.orchid.ac` | `http://127.0.0.1:8000` |

The SDK appends **`/v1/messages`** to this value, so the effective endpoint is `https://llm.orchid.ac/v1/messages`. If you mistakenly set `https://llm.orchid.ac/v1` as the Anthropic `base_url`, the client will call **`/v1/v1/messages`** and return **404**.

### OpenAI-compatible routes (`/v1/chat/completions`, `/v1/responses`, `/v1/models`, ‚Ä¶)

| Header | Value |
|--------|--------|
| `Authorization` | `Bearer <orchid-‚Ä¶>` |
| `Content-Type` | `application/json` (for POST bodies) |

The same string many clients call `api_key` is sent as the Bearer token.

### Anthropic-compatible route (`POST /v1/messages`)

| Header | Value |
|--------|--------|
| `x-api-key` | `<orchid-‚Ä¶>` (Anthropic-style; typical for the Anthropic SDK) |
| `anthropic-version` | `2023-06-01` (**required** on `/v1/messages`) |
| `Content-Type` | `application/json` |

`Authorization: Bearer <orchid-‚Ä¶>` is also accepted with the **same** Orchid key if your client prefers it.

See [Messages API](/api-reference/messages) for optional Orchid headers (`x-orchid-dehallucinate`, `x-orchid-thinking`, ‚Ä¶) on this route.

## Environment variables

```bash
export ORCHID_API_KEY="orchid-your-key-here"
```

Then in code (OpenAI client ‚Äî **include `/v1`** in `base_url` / `baseURL`):

<CodeGroup>

```python Python
import os
from openai import OpenAI

client = OpenAI(
    base_url="https://llm.orchid.ac/v1",
    api_key=os.environ["ORCHID_API_KEY"],
)
```

```typescript TypeScript
const client = new OpenAI({
  baseURL: "https://llm.orchid.ac/v1",
  apiKey: process.env.ORCHID_API_KEY!,
});
```

</CodeGroup>

## Error responses

Most gateway errors (including auth and rate limits) return an **OpenAI-compatible** JSON object with an `error` field (`message`, `type`, `code`). That applies to **`/v1/chat/completions`** and typically to **`/v1/messages`** as well, so Anthropic SDK users may see failures that do not deserialize as native Anthropic error types‚Äîuse logs or raw HTTP bodies when debugging.

```json
{
  "error": {
    "message": "Invalid API key",
    "type": "authentication_error",
    "code": "invalid_api_key"
  }
}
```

| Status | Meaning |
|--------|---------|
| `401` | Missing or invalid API key |
| `403` | Key exists but is inactive |

## Troubleshooting

| Issue | What to check |
|--------|----------------|
| **401** | `Authorization: Bearer ‚Ä¶` or `x-api-key` present and matches an active key in the dashboard. |
| **404, response body** `{"detail":"Not Found"}` **on** `/v1/messages` | The Anthropic `base_url` (or `ANTHROPIC_BASE_URL`) **includes an extra** `/v1`. Use `https://llm.orchid.ac` with **no** `/v1` so the client does not call `/v1/v1/messages`. |
| **404 on OpenAI ‚Äúbase URL‚Äù** | OpenAI-style URL must include **`/v1`** (for example `https://llm.orchid.ac/v1`, not the bare host) for clients that do not add `/v1` themselves. |
| **Model errors** | Body field **`model`** must be **`orchid01`** (see [GET /v1/models](/api-reference/models)). |
| **Messages / Anthropic SDK** | Send **`anthropic-version: 2023-06-01`**. Without it, the request is rejected. |
| **Timeouts** | Large `max_tokens` or slow upstream‚Äîretry with backoff or reduce output size. See [Rate limits](/api-reference/rate-limits). |
