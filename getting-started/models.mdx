## orchid01

orchid01 is Orchid's finance-native model. Use model id **`orchid01`** in all requests. The catalog is also available from [GET /v1/models](/api-reference/models).

There are two modes — standard and thinking — controlled via the **`orchid`** object on **`/v1/chat/completions`**, or via headers on **`/v1/messages`** (see below).

## Standard mode (default)

Fast, direct responses. Recommended for most tasks: document extraction, summarisation, Q&A over provided context, structured data conversion.

```python
response = client.chat.completions.create(
    model="orchid01",
    messages=[{"role": "user", "content": "Extract all covenants from this agreement..."}],
    # thinking is off by default
)
```

| Setting | Value |
|---------|-------|
| Default `max_tokens` | `4096` |
| Default `temperature` | `0.1` |
| Recommended `max_tokens` | `4096–8192` |

## Thinking mode

Extended reasoning for complex multi-step analysis. The model reasons through the problem before responding. Reasoning is visible in `reasoning_content` alongside the final answer in `content`.

```python
response = client.chat.completions.create(
    model="orchid01",
    messages=[{"role": "user", "content": "Build a full covenant analysis..."}],
    extra_body={"orchid": {"thinking": True}},
)

# Access reasoning and answer separately
reasoning = response.choices[0].message.model_extra.get("reasoning_content", "")
answer    = response.choices[0].message.content
```

<Warning>
  Thinking mode requires `max_tokens` ≥ 16,000. Reasoning tokens count toward the limit — responses may be truncated below this threshold.
</Warning>

| Setting | Value |
|---------|-------|
| Minimum `max_tokens` | `16000` |
| Recommended `max_tokens` | `16384–32768` |
| `temperature` | Fixed at `1.0` (set automatically) |

<Info>
  If you pass `max_tokens` below 16,000 with thinking enabled, Orchid automatically raises it to 16,000 and includes `max_tokens_adjusted: true` in the response metadata.
</Info>

## The `orchid` config object (`/v1/chat/completions` and `/v1/responses`)

Pass an `orchid` key alongside standard OpenAI fields — the field is identical on both endpoints (see [Responses](/api-reference/responses)):

```json
{
  "model": "orchid01",
  "messages": [...],
  "orchid": {
    "thinking":         false,
    "thinking_effort":  "medium",
    "dehallucinate":    false,
    "regenerate":       false
  }
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `thinking` | boolean | `false` | Enable deep reasoning mode |
| `thinking_effort` | string | — | `none`, `low`, `medium`, or `high` (with Fireworks reasoning) |
| `dehallucinate` | boolean | `false` | Set `true` to run the grounding judge on non-streaming completions |
| `regenerate` | boolean | `false` | Set `true` to auto-correct unsupported claims where possible. Implies `dehallucinate: true`. |

## Messages API (`POST /v1/messages`)

There is **no** `orchid` object in the JSON body. Use optional **headers** instead:

| Header | Values |
|--------|--------|
| `x-orchid-dehallucinate` | `true` or `false` |
| `x-orchid-regenerate` | `true` or `false` — implies `x-orchid-dehallucinate: true` |
| `x-orchid-thinking` | `true` or `false` |
| `x-orchid-thinking-effort` | `none`, `low`, `medium`, or `high` |

User-defined tool-use and image content blocks are not supported on the Messages façade yet — use [`/v1/chat/completions`](/api-reference/chat-completions) or [`/v1/responses`](/api-reference/responses) for those. The built-in `orchid-finance` tool suite still runs automatically on every endpoint, including Messages — see [Tool calling](/guides/tool-calling). Full route details: [Messages API](/api-reference/messages).

## Response metadata

Every response includes an `orchid` field — the `mcp_*` audit fields are always present, and the grounding fields appear with their documented values when the judge ran:

```json
{
  "choices": [...],
  "orchid": {
    "mcp_server":              "orchid-finance",
    "mcp_iterations":          1,
    "mcp_tool_calls":          [],
    "dehallucinate_requested": true,
    "grounded":                true,
    "score":                   0.97,
    "flagged_spans":           [],
    "checked":                 true
  }
}
```

| Field | Description |
|-------|-------------|
| `mcp_server` / `mcp_iterations` / `mcp_tool_calls` | Built-in tool-use audit trail — see [Tool calling](/guides/tool-calling#mcp-audit-fields) |
| `grounded` | Whether all claims are supported by provided context |
| `score` | Confidence score 0.0–1.0 (1.0 = fully grounded) |
| `flagged_spans` | Specific claims that could not be verified |
| `checked` | Whether the grounding check actually ran |
| `span_fixes` | Present when `regenerate` ran — see [Hallucination detection](/guides/hallucination-detection#auto-correcting-with-regenerate) |

The grounding check runs when **`dehallucinate`/`regenerate`** (chat, responses) or **`x-orchid-dehallucinate`/`x-orchid-regenerate`** (Messages) is `true`, the response contains checkable claims, and sufficient context was provided. If skipped, `checked: false` and `score: 1.0`. Full details: [Hallucination detection](/guides/hallucination-detection).
