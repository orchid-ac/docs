## Setup

Get your API key from the [Orchid dashboard](https://app.orchid.ac) under **Settings → API Keys**.

<Info>
  The API is **OpenAI-compatible** (`/v1/chat/completions`, `/v1/responses`, `/v1/models`, …) and **Anthropic-compatible** (`POST /v1/messages`). Use your Orchid key. **OpenAI-style** clients use `https://llm.orchid.ac/v1`. **Anthropic** clients and [Claude Code](/integrations/claude-code) use the **host only** — `https://llm.orchid.ac` — with **no** `/v1` on the base URL (the SDK appends `/v1/messages`). See [Authentication](/getting-started/authentication) for headers and troubleshooting.
</Info>

## Your first call

<CodeGroup>

```python Python
from openai import OpenAI

client = OpenAI(
    base_url="https://llm.orchid.ac/v1",
    api_key="orchid-your-key-here",
)

r = client.chat.completions.create(
    model="orchid01",
    messages=[{"role": "user", "content": "Summarise this 10-K filing..."}],
)
print(r.choices[0].message.content)
```

```typescript TypeScript
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://llm.orchid.ac/v1",
  apiKey: process.env.ORCHID_API_KEY!,
});

const r = await client.chat.completions.create({
  model: "orchid01",
  messages: [{ role: "user", content: "Summarise this 10-K filing..." }],
});
console.log(r.choices[0]?.message?.content);
```

```bash curl
curl -s https://llm.orchid.ac/v1/chat/completions \
  -H "Authorization: Bearer orchid-your-key-here" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "orchid01",
    "messages": [{"role": "user", "content": "Summarise this 10-K filing..."}]
  }'
```

</CodeGroup>

## Anthropic Messages (Claude Code and Python SDK)

For **`POST /v1/messages`**, including **Claude Code** and the official Anthropic Python client, the base URL must be **`https://llm.orchid.ac` without** `/v1` — the SDK appends `/v1/messages`. Setup, environment variables, and code samples: [Claude Code](/integrations/claude-code). Route reference: [Messages API](/api-reference/messages).

User-defined tool-use and images on **`/v1/messages`** are not supported yet — use **chat completions** or **[responses](/api-reference/responses)** for tools ([Tool calling](/guides/tool-calling)). The built-in finance tool suite still runs automatically on every endpoint, including Messages.

## Streaming

<CodeGroup>

```python Python
stream = client.chat.completions.create(
    model="orchid01",
    messages=[{"role": "user", "content": "Analyse this credit agreement..."}],
    stream=True,
)
for chunk in stream:
    piece = chunk.choices[0].delta.content or ""
    if piece:
        print(piece, end="", flush=True)
```

```typescript TypeScript
const stream = await client.chat.completions.create({
  model: "orchid01",
  messages: [{ role: "user", content: "Analyse this credit agreement..." }],
  stream: true,
});
for await (const chunk of stream) {
  process.stdout.write(chunk.choices[0]?.delta?.content || "");
}
```

</CodeGroup>
